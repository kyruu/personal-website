COVID-19 Chatbot

Overview:

Developed IBM Watson Assistant chatbot for the 2020 Call for Code initiative in an effort to help communities fight the pandemic. The chatbot leveraged data from The Weather Company‚Äôs Data Disease Tracker API as well as hospital statistics from Johns Hopkins Center for Systems Science and Engineering to answer both qualitative and quantitative questions about the virus. We first created a Watson Assistant service instance, developed Node.js webhooks to connect the chatbot to external data sources and deployed the app to IBM Cloud.

Motive:

During my time as an IBM Summit Intern, I was itching for the opportunity to take on more technical tasks. Shortly after finding out about the company-wide intern hackathon, I immediately reached out to a few of my fellow interns to form a team. We entered the competition as Team Barbara (named after our awesome Boston Hub leader, of course) and got to work.

Execution:

We first created a Watson Assistant service instance and added a dialog skill to it, representing the steps we wanted the chatbot to follow when prompted with a certain type of greeting. A dialog skill consisted of three main components: intents, entities, and dialog. An intent represented the purpose of the user‚Äôs query, such as a question about the number of confirmed COVID-19 cases in Florida. To train the dialog skill to recognize our intents, we supplied it with many examples of user input (i.e. ‚Äúcases‚Äù, ‚Äúdeaths‚Äù, etc.) and indicated which intents they mapped to (i.e. ‚ÄúCOVID cases‚Äù). An entity represented a term that is relevant to an intent and provided a specific context for it. For example, we trained the skill to recognize cities and states for which a user was seeking COVID-19 data about. Lastly, the dialog represented the flow of conversation and dictated how the bot responded to familiar intents and entities.

Next step involved integrating the chatbot with external data sources. We utilized webhooks to trigger a request to one of our APIs once a certain intent was recognized. The webhook, written in Node.js, collected specified data from the dialog and stored it in context variables. It then sent this data as part of a HTTP POST request to either the Disease Tracker API or John Hopkins API, depending on the intent. The webhook would then parse relevant information from the JSON response object and respond to the user with its findings.

Finally, we deployed our chatbot to IBM Cloud as a Cloud Foundry Application and marveled at the Barbara Bot üòä.
